Overview

This document is an abstract test suite (ATS): a compendium of test assertions applicable to all implementations of the Web Coverage Server (WCS) 1.1.1 specification. An ATS provides a basis for developing an executable test suite (ETS) to verify that the implementation under test (IUT) conforms to all relevant functional specifications.

The abstract test cases (assertions) are organized into test groups that correspond to sets of capabilities. The following test groups are defined:

  1. WCS-Shared – Shared Operational Aspects
  2. WCS-Get Capabilities – GetCapabilities Request andResponse
  3. WCS-Describe Coverage – Describe Coverage Request and Response
  4. WCS-GetCoverage – Identifier, BoundingBox,TemporalSubset, Field Identifier, Interpolation Method Type, AxisSubset, RangeSubset KVP syntax, Output Format and store.
Limitations

There are some limitations and conditions to the ATS:

  • Only the WCS server is tested, not the client
  • The assertions are not classified into conformance classes in this revision of the ATS. The conformance classes that would be used are in OGC 07-058r3a. These conformance classes can draw on many subclauses and parts of subclauses. For that reason, each test case is written independently of the other test cases. A conformance class may be tested by performing a specified set of individual test cases.
  • Only the protocol is tested, not the data. Data testing awaits a WCS 1.1.1 data encoding profile. The coverage returned by a GetCoverage request is not tested. There is no test data.
  • All WCS servers implement HTTP GET GetCapabilities request using KVP encoding (OGC 07-067r2 Subclause 8.2.1). XML and SOAP encodings are also possible for GetCapabilities. KVP, XML and SOAP encodings are possible for DescribeCoverage and GetCoverage. GetCapabilities executable tests will be encoded in KVP. Other executable tests will be encoded in KVP, XML or SOAP as supported by the server.
  • The requests and responses in the tests when transformed to an executable test suite (ETS), conform to the WCS XML schemas available at http://schemas.opengis.net/wcs/1.1.1 (not ready as of 2008-02-27). Instead they can be found at http://test.schemas.opengis.net/wcs111/wcs111.zip. The schemas for ows can be found at http://schemas.opengis.net/ows/1.1.0.
  • It is assumed that the highest protocol version that the server supports is 1.1.1. Some tests in the tables ranging from 1 to 10 would have to be modified to include AcceptVersions if this is not true.
  • For the sake of simplicity, the GetCoverage request test cases will largely be written with only one parameter being tested at once. That is, only one parameter at a time will vary from the minimal GetCoverage request. To do otherwise would entail having too many tests.
  • ETS encoding transformations: When transforming the structure names to KVP the DescribeCoverage identifier becomes identifiers.
  • The Output and GridCRS are not tested.
